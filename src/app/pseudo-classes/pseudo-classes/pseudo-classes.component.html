PSEUDO-CLASSES 9/5/17

The motivation behind this exercise was to get some clarity on exactly when the :hover, :link, :focus, and :active pseudo-classes are applied to anchor elements when using a web app.

I've found the following to be true in Google Chrome on a desktop. Different browsers and devices may yield different results:

  1. :link pseudo-class is applied to all anchor elements that have an href attribute and value. Hovering over a link or clicking a link does not cause the :link pseudo-class to be removed.
  2. :hover pseudo-class is applied when a user hovers over a link, as continues to stay applied while the user clicks the link.
  3. :active pseudo-classes is applied only when the user has clicked the link, and has not yet released their click.
  4. :focus pseudo-classes is applied when the user clicks a link. It remains until the user clicks outside of the link or gives focus to another element.
CHECK ME OUT

The link above routes to a dummy component. Next to the link you'll find each pseudo-class that's applied to the link at any given time. Observe that the :link pseudo-class is applied at all times. Hover over the link, to observe that the :link and :hover pseudo-classes are applied at the same time. Also try clicking and holding down to observe that the :link, :hover, :active, and :focus classes are all applied simultaneously! Release the mouse and watch the :active pseudo-class disappear. Move your cursor outside of the link to watch :hover disappear. Click the body of the document to see the :focus pseudo-class dropped as well.

The pseudo-classes applied to the link at any given time a rendered entirely using CSS:

a:link::after { content: ":link"; } a:link:hover::after { content: ":link, :hover"; } a:link:hover:active:focus::after { content: ":link, :hover, :active, :focus"; } a:link:hover:focus::after { content: ":link, :hover, :focus"; } a:link:focus::after { content: ":link, :focus"; }
